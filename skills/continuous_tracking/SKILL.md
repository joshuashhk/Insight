# 持续跟踪报告

## 概述

以财报分析报告为基础，纳入K线走势、估值指标、近期公告、网络搜索等新增信息，生成市场当下的主要叙事逻辑和关注点。

## 前置条件

- 已完成财报分析报告（`stock/{股票代码}/memory/财报分析报告.md` 存在）

## 执行方式

整个工作流（步骤0-3）由主 agent 启动 subagent 执行，prompt 模板：

```
你是Insight金融分析系统的报告撰写agent。
任务：为{股票代码}({公司名称})生成持续跟踪报告。
严格按照 skills/continuous_tracking/SKILL.md 中的完整工作流执行。
必读：notes/source-reliability.md、notes/timeline-system.md
```

Subagent 不继承主对话内容，信息搜集阶段不受 Joshua 当前立场影响（搜索结果客观呈现，而非验证既有判断）。

**关于 thesis.md**：当 `stock/{股票代码}/thesis.md` 存在时，subagent **不读取**该文件——thesis 是持续跟踪完成后、经讨论更新的对象，不是信息搜集的输入。步骤0只读研究笔记中的"待验证/仍未知"条目用于聚焦搜索。

## 工作流

### 步骤0：回顾研究笔记与时间轴

读取 `stock/{股票代码}/memory/研究笔记.md`（如存在），重点关注"待验证"和"仍未知"条目，在后续搜索和撰写中有意识地寻找这些问题的答案。如发现相关证据，在报告中提及，但不直接修改研究笔记——状态更新留待讨论后进行。

读取 `stock/{股票代码}/memory/时间轴.md`（如存在），建立时间参照系。后续搜索到新信息时，先定位到时间轴上，按获取日期规则决定是否更新。如时间轴不存在，在步骤2.5中基于搜索结果补建。

### 步骤1：获取最新数据

```bash
python3 skills/continuous_tracking/fetch_data.py <股票代码>
```

输出到终端：近6个月K线数据（区间统计+周K线+大幅波动日）、当前估值数据（PE/PB/市值及5年分位）、近6个月重要公告（按类别分组）。

### 步骤1.5：确定本次跟踪焦点

在搜索前，综合研究笔记（如有）、步骤1数据、财报分析报告，显式输出本次跟踪的优先关注方向：

```
### 本次跟踪焦点
1. [焦点1] → 搜索方向：[具体搜索什么]
2. [焦点2] → 搜索方向：[具体搜索什么]
3. [焦点3] → 搜索方向：[具体搜索什么]

### 反向焦点
- 如果当前主流判断是错的，原因最可能是什么？
- 搜索方向：[主动寻找的否定性证据]
```

焦点来源（按优先级）：
- 研究笔记中的待验证问题（如有）
- 步骤1数据中发现的异常（大幅波动、估值变化等）
- 财报分析报告中标记的关注点（如"后续跟踪重点"）

**目的**：让搜索有的放矢，避免漫无目的的泛搜索。**反向焦点是必填项**——持续跟踪的对象往往是Joshua关注的标的，Claude有系统性偏向乐观方向的风险，反向焦点强制在搜索阶段就主动寻找否定性证据。本次跟踪焦点不写入最终报告，仅作为搜索和撰写的内部导航。

### 步骤2：网络搜索

根据步骤1.5确定的焦点进行搜索。同一批次内的搜索并行发出以减少往返。

**批次A（并行）——公司层面**

通用搜索 + 英文搜索（如公司有国际业务或所在行业有全球定价属性）：
- `{公司名称} 最新消息`
- `{公司名称} 研报`
- `{公司名称} 业绩 预期`
- `{英文公司名} outlook`（如适用）

**批次B（并行）——公告定向**

针对步骤1中高优先级公告（资本运作、监管问询等）进行定向搜索：
- `{公司名称} {公告关键词1}`（如"芯导科技 重大资产重组"）
- `{公司名称} {公告关键词2}`（如有多个高优先级公告）

**批次C（并行）——行业层面**

当批次A结果稀少（市场关注度低），或所在行业有全球定价属性/明确国际对标公司时触发。搜索词从财报分析报告第2章"本行业分析框架"中提取：
- `{行业关键词} 行业 趋势`（如"功率半导体 市场 价格"）
- `{主要竞品/同业公司} 业绩`
- `{英文行业名} market trend`（如适用）
- `{国际对标公司} earnings`（如适用）

市场关注度低本身是重要信息，应如实反映在报告中，不硬凑内容。

### 步骤2.5：搜索结果整理（检查点）

**分类标注**：将搜索结果按以下类别标注后再用于撰写：
- **事实**：已发生的事件、已披露的数据（如"Q3营收同比增长20%"）
- **观点**：分析师或市场参与者的判断（如"看好国产替代加速"）
- **数据**：可量化的指标或预测（如"券商一致预期2025年EPS 0.8元"）

标注来源和时间，在报告中引用时据此区分"公司披露"、"市场观点"和"市场数据"。

**时效性筛选**：根据信息发布时间决定引用方式：
- **近6个月**：正常引用
- **6-12个月**：可引用，但须标注时间，提醒读者注意时效
- **超过12个月**：原则上不引用，除非是仍在持续影响的重大事件（如未完成的重组）

**更新时间轴**：将搜索中发现的新事件更新到 `stock/{股票代码}/memory/时间轴.md`，遵循获取日期规则（详见 `notes/timeline-system.md`）。如时间轴不存在则新建。

**不利证据汇总**：搜索结果中有哪些内容与现有投资判断或当前主流叙事相悖？在搜索整理文件中单独列出，不可省略（没有则写"无"）。这是空方论据的原材料，第3章多空辩论中空方部分应覆盖这里列出的内容。

**写入检查点**：整理完成后，将标注后的搜索结果写入 `stock/{股票代码}/cache/搜索整理_YYYYMMDD.md`，告知 Joshua："搜索结果已整理保存"。

### 步骤3：撰写持续跟踪报告

阅读财报分析报告 + 步骤1数据 + 步骤2.5搜索结果整理（含标注），按下方报告结构撰写。

---

## 报告结构（4章）

逻辑链：**发生了什么 → 市场信息汇总 → 围绕焦点辩论 → 接下来关注什么**

### 第1章：走势与估值

**数据来源**：K线数据 + 估值数据（步骤1输出）

**输出模板**：
```
## 1. 走势与估值

### 区间表现
[近6个月股价整体涨跌幅、期间最高/最低价及对应日期]

### 走势阶段划分
- [阶段1: 日期范围] - [涨跌幅] - [走势特征及可能原因]
- [阶段2: 日期范围] - [涨跌幅] - [走势特征及可能原因]

### 成交量特征
[量价配合情况、放量/缩量关键节点]

### 关键价格位
| 指标 | 价格 | 说明 |
|------|------|------|
| 6个月最高 | XX | YYYY-MM-DD |
| 6个月最低 | XX | YYYY-MM-DD |
| 当前价格 | XX | YYYY-MM-DD |
| 6个月均价 | XX | - |

### 当前估值
| 指标 | 当前值 | 5年分位 |
|------|--------|---------|
| PE(TTM) | XX | XX% |
| PB | XX | XX% |

### 估值变化
[当前估值在5年分位中的位置意味着什么？若有上次持续跟踪报告，对比估值变化]
```

**写作提示**：
1. 计算区间整体涨跌幅，标注最高价/最低价及日期
2. 将走势划分为2-4个阶段，描述每段特征
3. 分析成交量变化，识别放量/缩量关键节点
4. 呈现当前估值水平及历史分位；若有上次持续跟踪报告则对比变化，否则基于5年分位描述当前位置

### 第2章：市场信息

**数据来源**：重要公告（步骤1输出）、网络搜索结果、财报分析报告

**输出模板**：
```
## 2. 市场信息

### 近期热点事件
- [事件1]：[简述及市场反应]
- [事件2]：[简述及市场反应]

### 市场在讨论什么
[当前市场最关注的2-3个议题，概括性描述]

### 叙事变化
[市场叙事与公司自述（财报分析）有何差异？若有上次持续跟踪，叙事发生了什么变化？]
```

**写作提示**：
1. 近期热点事件：优先关注"业绩类"、"监管问询"、"资本运作"、"风险提示"等高优先级公告和市场事件，结合公告时间点分析市场反应（参考K线走势）
2. 市场在讨论什么：从热点事件和搜索结果中归纳市场当前聚焦的核心议题，只概括议题本身，不展开多空论证（留给第3章）
3. 叙事变化：对比市场观点与公司自述的差异；若有上次持续跟踪报告，识别叙事的变化

### 第3章：多空辩论

**数据来源**：第2章市场信息、网络搜索结果、财报分析报告

**输出模板**：
```
## 3. 多空辩论

### 议题1：[核心分歧问题，以问句形式]
**多方**：[论据]
**空方**：[论据]

### 议题2：[核心分歧问题，以问句形式]
**多方**：[论据]
**空方**：[论据]

### 待验证的问题
[辩论中暴露的信息缺口——目前无法判断多空谁对，需要等待哪些数据/事件来验证]
```

**写作提示**：
1. 从第2章的市场议题中提炼2-4个多空分歧最大的焦点，每个焦点以问句形式作为议题标题
2. 每个议题下多空对辩，论据紧贴具体事实和数据，避免泛泛而谈
3. 待验证的问题是辩论中暴露的信息缺口——当前证据不足以判断多空谁对，列出需要等待的关键数据点或事件

### 第4章：近期催化剂

**数据来源**：网络搜索结果、财报分析报告

**输出模板**：
```
## 4. 近期催化剂

### 确定性事件
| 事件 | 预计时间 | 潜在影响 |
|------|----------|----------|
| 财报发布 | XXXX-XX | XX |

### 潜在催化剂
[可能发生但时间不确定的事件]

### 风险事件
[可能产生负面影响的事件]
```

**写作提示**：列出确定性事件及预计时间，识别潜在催化剂，提示可能的风险事件。

---

## 写作要求

1. 聚焦"市场当前怎么看"，而非重复财报分析的内容
2. 明确标注信息来源和时间（如"根据2024年1月研报"、"据财联社1月15日报道"）
3. 区分事实与观点，引用时注明是"市场观点"还是"公司披露"
4. 关注边际变化，而非静态描述
5. 保持客观中立，不做投资判断——判断留给投资洞察

## 输出

- 格式：Markdown
- 开头包含标题、股票代码、报告日期、财报分析日期
- 各章节之间用分隔线 `---` 分隔
- 保存到 `stock/{股票代码}/memory/持续跟踪_YYYYMMDD.md`
