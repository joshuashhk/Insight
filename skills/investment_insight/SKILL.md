# 投资洞察

## 概述

将研究笔记中经过讨论验证的认知整理为**可对外发布的投资分析报告**。投资洞察不是 Claude 的独立分析，而是 Joshua 与 Claude 讨论后形成的共识的正式表达。

**与其他报告的关系**：
- 财报分析、持续跟踪 = 信息搜集和整理（Claude可独立完成）
- 研究笔记 = 内部工作底稿（讨论产物，含Tier 2来源、未验证假设、推测）
- **投资洞察 = 对外发布版**（结论有据可依、来源可追溯、判断有验证节点）

## 前置条件

- 已完成财报分析报告（必需）
- 已完成持续跟踪报告（必需）
- **研究笔记中有足够的讨论沉淀**（必需）——至少有经过讨论确认的核心投资逻辑和关键判断

**何时不应触发**：如果研究笔记只有初始条目、没有经过讨论打磨的认知，说明还不到写投资洞察的时候。先讨论，再写报告。

## 工作流

### 步骤1：阅读研究笔记和已有报告

读取 `stock/{股票代码}/memory/` 下的：
- 研究笔记.md（**核心输入**——投资洞察的内容主要来自这里）
- 财报分析报告.md（背景信息）
- 持续跟踪_YYYYMMDD.md（最新一份，市场信息和多空辩论）
- 时间轴.md（时间参照系）

**关键区分**：
- 研究笔记中"✅ 已验证"的条目 → 可以写入投资洞察
- 研究笔记中"待验证"或"仍未知"的条目 → 写入"待验证"部分，标注信息缺口
- 研究笔记中标注为 Tier 2 来源的信息 → 不直接作为结论，降级为"需关注"或寻找 Tier 1 来源确认

### 步骤1.5：确认可发布的核心内容

显式列出从研究笔记中提取的、可以对外发布的核心判断：

```
### 可发布的核心判断
1. [判断1]（来源：研究笔记 YYYY-MM-DD，状态：已验证）
2. [判断2]（来源：研究笔记 YYYY-MM-DD，状态：已验证）
3. [判断3]（来源：研究笔记 YYYY-MM-DD，状态：待验证——需降级处理）
```

**目的**：确保投资洞察的每个核心观点都能追溯到研究笔记中的讨论，不是 Claude 自己编的。

### 步骤1.6：魔鬼代言人（内部压力测试，不进入最终报告）

针对步骤1.5列出的每个核心判断，假设最聪明的反对方来攻击——他们会怎么说？

这是思考过程，不是输出环节。反论的作用是修正判断本身：
- 找到有力反论 → **直接调整判断**（降低确定性、增加限定条件、修改结论），调整后的判断进入报告
- 找不到有力反论 → 判断更坚固，可能是逻辑确实强，**也可能是视野有盲区**——两种情况都要告知 Joshua

读者看到的应该是经过压力测试后的结论，不是辩论过程。

### 步骤2：补充搜索（可选）

仅在以下情况触发：
- 研究笔记中的待验证条目需要最新数据确认
- 距上次持续跟踪已有较长时间，需要更新市场信息
- Joshua 明确要求补充某方面信息

搜索结果用于补充和更新，不用于产生新的核心判断。

**数据来源优先级**（同持续跟踪）：
1. 英文权威来源：Bloomberg、Reuters、公司IR、国际投行研报
2. 行业数据库：Yahoo Finance、TrendForce、Gartner
3. 中文来源：财联社、证券时报、国内券商研报

### 步骤3：撰写投资洞察

基于步骤1的研究笔记和报告，按下方报告结构撰写，控制在约2000-3000字。

---

## 核心原则

1. **有据可依**：每个核心判断都能追溯到研究笔记中的讨论或可验证的数据来源
2. **深度 > 广度**：专注经过讨论验证的核心逻辑，不堆砌未经讨论的泛泛分析
3. **区分确定性**：明确标注哪些是已验证的判断、哪些是待验证的假设
4. **对外可读**：不暴露内部工作底稿的细节（Tier 2来源的原始推测、未成形的假设等），呈现的是经过提炼的结论

**禁止**：
- 重复财报分析或持续跟踪中已有的数据和描述
- 将研究笔记中标注为"待验证"的 Tier 2 信息当作已确认结论发布
- 没有讨论基础的"独立发现"——如果某个观点不在研究笔记中，它不应该成为投资洞察的核心判断

---

## 报告结构（5节）

### 第1节：核心投资逻辑（200-400字）

回答三个本质问题：
- **公司本质**：这家公司的本质是什么？（不是业务描述，而是商业模式本质）
- **盈利驱动**：核心利润来自哪里？驱动力是什么？脆弱性在哪里？
- **护城河**：真正的壁垒在哪里？有多宽？

以**关键矛盾**收尾：点出当前最核心的投资张力。

**要求**：
- 用第一性原理思考，超越表面业务描述
- 核心逻辑应来自研究笔记中经过讨论的认知框架
- 关键矛盾要一针见血

### 第2节：关键判断（核心内容）

呈现经过压力测试的2-3个核心判断。每个判断包含：
- **一句话结论**：直接表明立场
- **支撑逻辑**：为什么得出这个结论（简洁的推理链）
- **证据**：支撑这一判断的关键数据（标注来源层级）

判断本身应已吸收魔鬼代言人环节的压力测试结果——如果反论有效，判断的确定性和措辞应已调整；如果反论不成立，判断更坚固，无需在报告中展示被否定的反论。

**格式示例**：

**判断1：[一句话结论]**

[支撑逻辑，2-3句话]

证据：[关键数据点，标注来源]

**注意**：与持续跟踪第3章"多空辩论"的区别——多空辩论客观呈现市场两方观点，这里是我们经过讨论后选择的立场和理由。

### 第3节：待验证假设

列出尚未确认但对投资判断有重大影响的假设，每条标注：
- 当前状态（来源层级、可信度）
- 验证方向（需要什么数据/事件来确认）
- 影响程度（如果被证实/证伪，对投资逻辑的影响）

### 第4节：估值与验证节点

**当前立场**：一句话表明观望/看多/看空

**合理估值区间**：
- 给出估值区间及关键假设（不超过2个场景）

**验证节点**（2-4个，按时间排列）：
每个节点包含具体时间、观察事件、判断阈值。

**示例**：
1. **2026年2月底 Nvidia Q4财报**：数据中心收入增速>60%且指引积极→利好，指引保守→利空
2. **2026年4月 公司年报**：Q4毛利率>33%→验证基本面，<30%→下调预期

### 第5节：数据来源

列出本次分析引用的关键信息来源，按优先级分层：
- **Tier 1**（官方/权威）：公司IR、Bloomberg、Reuters、国际投行研报
- **Tier 2**（行业/专业）：Digitimes、Yahoo Finance、TrendForce
- **Tier 3**（中文/参考）：财联社、证券时报、国内券商研报

## 与研究笔记的关系

**研究笔记是投资洞察的上游，不是下游。**

```
财报分析 → 持续跟踪 → Joshua与Claude讨论 → 研究笔记 → 投资洞察
```

- 研究笔记记录讨论过程中的认知演进（含推测、待验证假设、Tier 2信息）
- 投资洞察将研究笔记中已验证的认知提炼为可发布的形式
- 投资洞察完成后，不反向写入研究笔记
- 如果投资洞察撰写过程中发现新问题，标记为"待讨论"，留给下次讨论
