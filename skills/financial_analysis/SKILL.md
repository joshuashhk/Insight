# 财报分析报告

## 概述

通过公司多年公开报告（年报、中报、招股说明书等）和财务数据，分析公司及其所在行业的基本特点，为进一步分析准备材料。

**材料局限性**：财报分析的所有材料均来自公司自身披露（财报、招股书等），本质上是公司管理层的单方面叙述。行业数据、竞争格局、市场地位等信息均为公司自述，未经第三方交叉验证。报告中应如实反映这一局限，避免将公司自述当作客观事实。在三层报告体系中，持续跟踪将引入市场观点与公司自述进行对比，投资洞察则在此基础上形成独立判断。

## 执行方式

**步骤1-2** 由主 agent 直接执行（涉及网络下载，避免 subagent 超时）。

**步骤3-4** 由主 agent 启动 subagent 执行，prompt 模板：

```
你是Insight金融分析系统的报告撰写agent。
任务：为{股票代码}({公司名称})生成财报分析报告。
步骤1-2已由主agent完成，stock/{股票代码}/memory/ 和 stock/{股票代码}/filings/ 下的材料已就绪。
请执行步骤3（建立时间轴）和步骤4（撰写报告），严格按照 skills/financial_analysis/SKILL.md 中的格式要求。
必读：notes/analysis-philosophy.md、notes/timeline-system.md
```

Subagent 只看到 SKILL.md，不继承主对话内容。这同时解决两个问题：（1）防止长报告写作压缩主 agent context；（2）确保行业分析框架、隐含风险等判断不受 Joshua 既有立场影响。

## 工作流

### 步骤1：提取公司信息

```bash
python3 skills/financial_analysis/export_company_info.py <股票代码>
```

生成 `stock/{股票代码}/memory/公司信息.json`，包含：

| 字段 | 内容 |
|------|------|
| 基础信息 | 股票代码、名称、上市日期、所属指数 |
| 公司概况 | 管理层、主营业务、前十大股东 |
| 资产负债表 | 近8季度+5年年报（单位：亿元） |
| 利润表 | 近8季度+5年年报（单位：亿元） |
| 现金流量表 | 近8季度+5年年报（单位：亿元） |
| 基本面数据 | PE、PB、ROE等估值与基本面指标 |

### 步骤2：下载公告并提取关键章节

```bash
python3 skills/financial_analysis/fetch_announcements.py <股票代码>
```

下载年报、中报、招股说明书等PDF，自动提取关键章节为Markdown。

下载源：
- 深交所(SZ)股票：直接从深交所下载
- 上交所(SH)股票：通过巨潮资讯(cninfo)下载

提取规则（避免与JSON重复）：
- **年报/中报**：仅提取"管理层讨论与分析"（完整提取，含主要经营情况）
- **招股说明书**：业务与技术 + 风险因素 + 募集资金运用
- **增发/可转债**：募集资金使用 + 风险因素（仅保留最终版）
- **中报**：仅下载最新一期，且对应年份无年报时才下载

输出：
```
stock/{股票代码}/
├── memory/
│   ├── 年报摘要.md          # 3年，管理层讨论与分析
│   ├── 中报摘要.md          # 仅无对应年报的期数
│   ├── 招股说明书摘要.md（如有）
│   ├── 增发摘要.md（如有）  # 同一次募资仅最终版
│   └── 可转债摘要.md（如有）
└── filings/
    ├── 年报/
    ├── 中报/
    └── ...
```

### 步骤3：建立时间轴

阅读步骤1-2的材料，提取关键历史事件和已知的预期事件，建立 `stock/{股票代码}/memory/时间轴.md`。格式和规则见 `notes/timeline-system.md`。

### 步骤4：撰写财报分析报告（分阶段写入）

阅读 `stock/{股票代码}/memory/` 下所有材料（公司信息.json + 各摘要.md），**分两个阶段**撰写报告，每阶段完成后立即写入文件。

**阶段A**：撰写第1-2章 → 写入 `stock/{股票代码}/memory/财报分析报告.md`（部分）→ 告知 Joshua："第1-2章已保存"

> 第2章的"本行业分析框架"是后续章节的核心锚点，必须在阶段A中确定并保存。

**阶段B**：撰写第3-5章 + 后续跟踪重点 → 追加到报告文件 → 告知 Joshua："报告完成"

---

## 报告结构（5章）

### 第1章：公司与业务

**数据来源**：公司信息.json（基础信息、公司概况、营收构成）+ 年报摘要 + 中报摘要 + 招股说明书摘要 + 增发摘要

**输出模板**：
```
## 1. 公司与业务

### 一句话定位
[公司是谁、做什么、在行业中什么位置]

### 主营业务构成
[各业务/产品线收入占比、毛利率、同比变化]

### 产品/服务详解
[核心产品、应用场景、下游客户画像]

### 商业模式
[怎么赚钱：收入确认方式、账期特征、现金流模式]

### 客户与供应商
[前五大客户/供应商占比、依赖度变化]

### 业务演进与战略方向
[近3年重大变化：新增/退出业务、并购、管理层战略解读、在建项目与产能规划]
```

**写作提示**：
1. 一句话定位要精准概括公司本质，不是泛泛的业务描述
2. 优先引用最新财报数据，对比历史数据分析变化趋势
3. 商业模式要解释清楚公司**怎么赚钱**，而不只是做什么
4. 业务演进要体现管理层战略思路的变化，而不只是罗列事件

---

### 第2章：行业分析

**数据来源**：招股说明书摘要（业务与技术章节）+ 年报摘要（管理层讨论与分析中的行业描述）

**输出模板**：
```
## 2. 行业分析

### 行业概况
[行业定义、产业链位置、市场规模及增速]

### 行业驱动因素
[需求端驱动、供给端变化、技术演进、政策影响]

### 竞争格局
[市场集中度、主要玩家、公司竞争地位]

### 行业趋势
[未来3-5年行业发展趋势、机会与威胁]

### 本行业分析框架
基于以上行业特性，后续章节应重点关注：
- **核心竞争力**：[本行业中，决定公司成败的关键能力是什么？如：技术壁垒/品牌渠道/成本控制/牌照资质/...]
- **财务关键指标**：[本行业最应关注的2-3个财务指标。如：毛利率趋势/研发费用率/产能利用率/同店增长/净息差/...]
- **重点风险类型**：[本行业特有的风险关注点。如：技术路线风险/原材料周期/政策变化/消费偏好转移/...]
```

**写作提示**：
1. 客观描述行业现状，不要过度乐观或悲观
2. 关注行业的周期性特征
3. **"本行业分析框架"是本章最重要的输出**——它直接决定第3、4、5章的分析方向。要根据行业特性给出具体、可操作的指引，不要泛泛而谈

---

### 第3章：核心竞争力

**数据来源**：年报摘要 + 招股说明书摘要 + 公司信息.json

**方向**：根据第2章"本行业分析框架"中确定的核心竞争力方向展开分析。

不同行业的侧重点不同，以下为参考：

| 行业类型 | 核心竞争力分析方向 |
|----------|------------------|
| 科技/芯片 | 研发投入与团队、技术壁垒、专利/IP、产品迭代能力 |
| 消费品 | 品牌力、渠道布局、供应链效率、消费者粘性 |
| 制造业 | 产能与利用率、成本控制能力、工艺壁垒、规模效应 |
| 金融 | 风控能力、牌照/资质、资产质量、客户基础 |
| 医药 | 研发管线、审批进度、销售能力、专利保护期 |

**输出模板**：
```
## 3. 核心竞争力

### [竞争力维度1]
[分析内容，用数据支撑]

### [竞争力维度2]
[分析内容，用数据支撑]

### 竞争力评估
[综合判断：壁垒高低、可持续性、薄弱环节]
```

**写作提示**：
1. 不要试图覆盖所有方面，聚焦第2章确定的2-3个关键维度
2. 每个维度用具体数据支撑，不要空谈
3. 竞争力评估要诚实——既指出优势，也指出薄弱环节

---

### 第4章：财务分析

**数据来源**：公司信息.json（核心）+ 年报摘要 + 中报摘要（管理层对财务数据的解读）

**方向**：根据第2章"本行业分析框架"中确定的财务关键指标展开分析。

**输出模板**：
```
## 4. 财务分析

### 行业关键指标（近5年趋势）
[根据第2章确定的2-3个行业关键指标进行深入分析，用表格呈现趋势]

### 成长性与盈利能力
- 营收/利润增速及CAGR
- 毛利率、净利率变化
- 四项费用率趋势（销售/管理/研发/财务）
- ROE趋势

所有数据只使用原始数字和计算结果，不引入外部估算或推算数据。说明各指标变动原因。

### 财务健康度
- 资产负债率、现金流质量
- 重大科目异常变化
```

**写作提示**：
1. **"行业关键指标"是本章最重要的部分**——不要用通用指标替代，要选第2章指明的行业特定指标
2. 不要面面俱到，聚焦最能反映公司质地的指标
3. 结合管理层解读理解变动原因，不要只列数字

---

### 第5章：风险因素

**数据来源**：招股说明书摘要（风险因素章节）+ 年报摘要 + 中报摘要（管理层讨论与分析中的风险提示）

**方向**：在第2章"本行业分析框架"中确定的重点风险类型基础上，结合公司披露的风险信息进行分析。

**输出模板**：
```
## 5. 风险因素

### 管理层披露的主要风险
[只保留3-5个实质性风险，排除套话和行业通用免责表述；每条说明为何认为是实质性的]

### 风险变化
[对比历年年报和招股书，新增/消除了哪些风险]

### 隐含风险
[管理层未明说但报表或业务中体现的问题]
```

**写作提示**：
1. "管理层披露的主要风险"：从所有披露风险中主动筛选，只写实质性的3-5条，不要罗列清单
2. 对比最新财报或招股书的风险披露，识别新增/消除的风险
3. 结合财务分析，识别管理层未明说但报表中体现的隐含风险
4. 重点关注第2章指出的行业特有风险类型

---

### 报告末尾：后续跟踪重点

在第5章之后，增加一节"后续跟踪重点"，为持续跟踪提供方向指引。

**输出模板**：
```
## 后续跟踪重点

基于本次分析，持续跟踪应优先关注：
1. [核心假设/悬而未决的问题] → 验证信号：[具体观察什么]
2. [核心假设/悬而未决的问题] → 验证信号：[具体观察什么]
3. [数据缺口/信息不足之处] → 信息来源建议：[去哪找]
```

**写作提示**：
1. 聚焦3-5个最关键的问题，不要罗列所有风险
2. 每个问题要配具体的验证信号或信息来源，确保可操作
3. 这一节是财报分析对持续跟踪的"交接清单"——当研究笔记尚不存在时，它是持续跟踪的主要方向指引

---

## 写作要求

1. 客观陈述事实，避免主观臆断
2. 仅标注原始来源（如"根据2024年年报"），不需要引用文件名
3. 使用具体数字，而非模糊描述
4. 分析要有深度，避免泛泛而谈
5. 优先引用发布日期最晚的一次财报数据

## 输出

- 格式：Markdown
- 开头格式固定为：
  ```
  # {公司名}（{股票代码}）财报分析报告

  **报告日期**：{YYYY-MM-DD}

  ---
  ```
- 各章节之间用分隔线 `---` 分隔
- 文末固定追加免责声明：
  ```
  ---

  **免责声明**：本报告由 Insight 分析系统生成，仅供个人学习和研究参考，不构成任何投资建议。报告内容基于公开披露信息整理，不保证准确性和完整性。作者可能持有报告中提及的证券。投资有风险，决策需谨慎。
  ```
- 保存到 `stock/{股票代码}/memory/财报分析报告.md`
