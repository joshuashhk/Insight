# 叙事审阅

## 概述

对卖方研究报告中的核心叙事进行批判性审阅。不提取信息，而是评估论证质量：区分事实与推论，显式化隐含假设，判断结论的支撑强度。

**前提**：已通过读研报积累了足够的领域知识。审阅的有效性取决于领域知识的深度——skill 提供框架，判断力来自积累。

**注意力的反转**：报告用大量篇幅呈现事实，但事实对结论的贡献往往有限。真正决定结论是否成立的，是那些未被说出口的假设——它们可能只占报告极小篇幅，甚至完全不出现，却承担了全部的论证重量。审阅时的注意力分配应与读报告时完全相反：重心在没有呈现的，而不是呈现出来的。

**与读研报的关系**：读研报是吸收，叙事审阅是批判。两件事不应同时做。叙事审阅是投资洞察的上游，先清理叙事质量，再形成自己的判断。

## 触发时机

- 某个题材已读多篇研报，准备形成判断之前
- 发现卖方市场空间测算数字差异极大，或结论与直觉有明显出入
- 准备写投资洞察之前

## 工作流

### 步骤1：定义被审阅的叙事

用1-2句话写出卖方的核心论断是什么、结论是什么。

### 步骤2：画出推论链

把从出发点到结论的逻辑链逐步写出来：

```
A → B → C → 结论
```

每个节点是一个具体命题，箭头是因果或推导关系。

粒度不是预设的——在感觉有跳跃的地方往下挖一层，直到把重要的隐含假设暴露出来为止。没有跳跃的地方不需要拆细。

### 步骤3：标注事实与推论

对每个节点标注性质：

- **事实**：有实测数据、真实事件、公开合同支撑
- **推论**：从其他事实推导，推导过程本身需要检验
- **假设**：没有直接支撑，默认成立的前提

### 步骤4：显式化隐含假设

在每个推论和假设节点，把使它成立所需的前提条件写出来：

> 要让「X」成立，需要假设「Y」。Y 的依据是什么？

目标是让所有隐藏在"→"里的东西变得可见。

### 步骤5：识别关键假设

结论成立要求所有假设同时为真，但结论失败只需要一个假设为假——验证与否定的效率极度不对称。因此不应平均分配注意力，而应先找到最脆弱的假设：

- **支撑证据最弱**：几乎没有独立依据
- **论证重量最大**：去掉它，结论直接崩塌

把精力集中在这个假设上。其余假设即便成立，关键假设不成立，结论已经错了。

### 步骤6：评估关键假设

对关键假设深入问：

- 它有没有独立的事实依据？
- 把它单独列出来，是否仍然看起来合理？
- 有没有相反的证据、或未被考虑的替代路径？

### 步骤7：输出审阅结论，讨论

## 输出格式

```markdown
## 叙事审阅：{题材}

**核心论断**：{卖方在说什么，结论是什么}

### 推论链

{A → B → C → 结论}

各节点性质：
- A：事实（{依据}）
- A→B：推论（{推导逻辑}）
- B：假设（{无独立依据}）
- ...

### 隐含假设清单

| 假设 | 独立依据 | 评估 |
|------|---------|------|
| {假设1} | 有/无/弱 | 合理/存疑/不成立 |
| {假设2} | ... | ... |

### 结构性判断

**有事实支撑的部分**：
- {列出}

**依赖未验证假设的部分**：
- {列出}

**结论成立还需要什么为真**：
- {列出关键缺失条件}
```

## 常见模式参考

以下是卖方叙事中高频出现的隐含假设类型，审阅时可对照：

| 模式 | 表现 | 隐藏的假设 |
|------|------|----------|
| 需求混淆 | 确定性小需求与想象性大需求合并为一个数字 | 第二个需求驱动会在预设规模下成立 |
| 参数混淆 | 不同环境下测量的参数并排比较 | 不同环境下的参数可以互相替代 |
| 数据可信度伪装 | 实验室/地面数据支撑太空/实战结论 | 推算环境等价于真实环境 |
| 逻辑链跳跃 | 因果链中存在未被排除的替代路径 | 这是A到B的唯一可能路径 |
| 真实事件背书 | 用技术里程碑支撑市场规模结论 | 技术可行等价于市场成立 |
| 情景锚定 | 保守/乐观情景并列，大数字主导讨论 | 乐观情景具有与保守情景同等的参考价值 |

这些模式的底层结构相同：**用可信的A为不可信的B建立信用，通过合并或省略让跳跃不可见。**
