# 搜索策略 设计笔记

## 搜索在持续跟踪中的角色 (2026-02-08)

**背景**：梳理持续跟踪报告结构时，分析了网络搜索对各章节的支撑作用，发现搜索策略需要根据公司的市场关注度自适应调整。

### 各章节对搜索的依赖程度

| 章节 | 依赖程度 | 说明 |
|------|----------|------|
| 1. 走势与估值 | 无 | 完全依赖结构化数据（K线+估值） |
| 2. 市场观点 | **重度** | 主流观点、多空逻辑、分歧焦点——只能从外部获取 |
| 3. 核心关注点 | 部分 | 公告提供线索，但事件内容和市场反应需搜索补充 |
| 4. 近期催化剂 | 部分 | 确定性事件可推算，潜在催化和风险需搜索 |

**核心发现**：搜索的主要价值是为第2章"市场观点"服务。第3、4章是补充。

### 两层搜索策略

**第一层：公司搜索**
- 焦点搜索：针对步骤1数据中发现的具体问题（大幅波动原因、公告后续等）
- 通用搜索：`最新消息`、`研报`、`业绩 预期`

**第二层：行业/同业搜索**（第一层结果稀少时启动）
- `{行业关键词} 行业 趋势`
- `{主要竞品/同业} 业绩`

### 市场关注度问题

这是搜索策略面临的核心挑战。

**高关注度公司**（如茅台）：搜索返回大量研报、媒体报道、投资者讨论，第2章素材充足，甚至需要筛选去噪。

**低关注度公司**（如芯导科技688230）：搜索结果稀少，可能只有零星新闻或交易所公告搬运。第2章难以写出实质内容。

**当前处理方式**：
- 市场关注度低本身是重要信息，如实反映在报告中
- 启动第二层搜索，从行业和同业角度间接推断公司面临的环境
- 不硬凑内容

### 待研究的问题

1. **搜索结果质量评估**：如何判断搜索结果的质量和可信度？研报 vs 自媒体 vs 论坛讨论，应该有不同的引用权重。目前没有机制区分。

2. ✅ **时效性控制**（2026-02-08）：在步骤2.5中加入时效性筛选——近6个月正常引用，6-12个月须标注时间，超12个月原则上不引用（除非仍在持续影响的重大事件）。不在搜索词层面硬控制（搜索引擎对时间词理解不精确），而是在整理环节过滤。

3. **同业搜索的公司选择**：当触发第二层搜索时，"主要竞品/同业"从哪里来？财报分析报告中的行业分析可能提到了竞争对手，但不一定系统化。是否需要在财报分析时就显式列出可比公司？

4. ✅ **搜索词行业自适应**（2026-02-08）：从财报分析报告第2章"本行业分析框架"中提取行业关键驱动因素作为搜索词，替代硬编码通用模板。已写入SKILL.md步骤2第二层搜索。

5. ✅ **公告定向搜索**（2026-02-08）：针对步骤1中高优先级公告（资本运作、监管问询等）提取核心关键词进行定向搜索，获取市场对该事件的反应。已写入SKILL.md步骤2第一层搜索。

6. ✅ **英文信息源**（2026-02-08）：选择性触发——有海外业务、全球定价行业、或有国际对标公司时才搜英文。作为第三层搜索写入SKILL.md。

7. ✅ **搜索结果结构化标注**（2026-02-08）：搜索结果在写入报告前按"事实/观点/数据"分类标注，并记录来源和时间。作为步骤2.5写入SKILL.md。
